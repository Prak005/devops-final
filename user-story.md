# User Stories

## User Story 1
**Title:** User Login

**As a** registered user  
**I want** to log in to the application using my credentials  
**So that** I can access the system's features securely.

### Acceptance Criteria
- User can enter username and password.
- System validates the credentials.
- Successful login redirects the user to the dashboard.
- Invalid credentials display an appropriate error message.

---

## User Story 2
**Title:** View Build Status

**As a** developer  
**I want** to view the build status of the project  
**So that** I can know whether the latest changes were built successfully.

### Acceptance Criteria
- Build status is visible in the project README.
- The status updates automatically after each build.
- Developers can identify build failures quickly.
